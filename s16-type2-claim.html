S16.0 — Type II 개요 및 TFL 유형 체계 내 위치

Type II 자기선언 신뢰성 사실 클레임은 AI 제품(시스템·모델·서비스)의 개발자 또는 공급자가 특정 신뢰성 특성에 대해 스스로 측정·검증하고, 그 결과를 사실(fact) 형태로 공개 선언하는 문서입니다. 제3자 인증 없이 가능하나, 모든 진술은 측정 가능하고 검증 가능한 구체적 사실에 근거해야 합니다.

S16.2 — 클레임 필수 요건 (ISO/IEC 42117-1 §7.1.2 기반)

§7.1.1에 따라 아래 17개 항목은 모든 신뢰성 사실 진술에 적용(SHALL)됩니다. Type II 클레임 작성 시 각 항목의 충족 여부를 체크해야 합니다.

§7.1.2 — 17개 진술 요건 체크리스트 (a~q)
# 항목 요건 내용 (SHALL) Type II 적용 S03 연계
1a)정확하고, 오해를 유발하지 않으며, 잘못된 해석을 초래할 가능성이 없어야 합니다필수모든 클레임
2b)입증 가능하고 검증 가능해야 합니다필수metric + actual_result 필드
3c)해당 특정 AI 시스템 또는 모델에 관련이 있어야 합니다필수product 필드
4d)진술이 전체 AI 시스템에 적용되는지 특정 모델·컴포넌트·데이터셋에만 적용되는지 명확히 표시해야 합니다필수applicable_to 필드
5e)진술된 신뢰성 특성 또는 신뢰성 개선에 대해 구체적이어야 합니다필수claim_text + threshold
6f)진술 개발 방법론(기준, 파라미터, 계산·테스트 방법, 벤치마크 데이터셋)에 대해 투명해야 합니다필수evidence.standard_ref
7g)단일 신뢰성 변화에 대해 다중 이익을 암시하기 위해 다른 용어로 재진술해서는 안 됩니다필수 (준수)특성 1개 = 클레임 1개
8h)AI 라이프사이클의 모든 관련 단계를 고려해야 합니다 (NOTE: 자기선언은 종종 한 단계만 다룸)적용·문서화lifecycle 필드
9i)실제 제3자 인증이 없을 때 제3자 인증으로 오인될 방식으로 표현해서는 안 됩니다필수claim_type: "Type II"
10j)존재하지 않는 신뢰성 개선을 암시하거나 신뢰성 특성을 과장해서는 안 됩니다필수actual_result 정확 기재
11k)문자적으로 사실이라도 오해될 가능성이 있거나 관련 사실의 누락으로 오도되는 경우 진술 금지필수limitations 필드
12l)존재하거나 실현될 가능성이 있는 AI 시스템·모델의 신뢰성 특성에만 관련되어야 합니다필수characteristic 선택 시 확인
13m)지원 정보가 있는 경우 이를 명확히 표시하고 쉽게 이용 가능한 위치를 안내해야 합니다해당 시 필수supporting_info.location
14n)비교 진술의 경우, 구체적이고 비교 근거를 명확히 해야 합니다 (§7.4 참조)비교 시 필수comparative 필드
15o)기존의 미공개 측면에 기반한 경우, 최근 수정이나 혁신으로 오인하게 하는 방식으로 표현 금지필수—
16p)해당 AI 시스템·모델 카테고리와 무관한 특성의 부재에 기반한 진술 금지 (예: 이미지 분류기에 "텍스트 편향 없음")필수characteristic 적합성 확인
17q)기술 변화, 경쟁 시스템, 기타 상황 변화를 반영하여 필요 시 재평가 및 업데이트해야 합니다유지관리 필수valid_until + 재심사 주기
⚠️
§7.3 일반적 신뢰성 진술 제한
"sustainability" 또는 "ethicality"와 연계하는 광범위한 신뢰성 진술은 §6 요건을 충족한 프로그램 또는 공개적으로 이용 가능한 합의 기반 문서 내에 포함된 경우에만 허용됩니다. ISO/IEC TS 42117-1은 현재 PWI 준비 단계이므로 "모든 신뢰성 요건을 충족한다"는 광범위한 주장은 불가합니다.
📝 설명 진술(Explanatory Statement) 의무화 강화 REC-S16-02 · ISO 14021 §5.6 대응

ISO 14021 §5.6에 따르면, 자기선언 클레임이 오해를 유발할 가능성이 있거나 주장의 근거가 복잡한 경우 추가 설명 진술을 반드시 첨부해야 합니다. TFL Type II 클레임에서 다음 조건 중 하나라도 해당하면 explanatory_statement 필드 작성이 의무입니다:

설명 진술 의무 조건 진술 내용 요건 ISO 14021 대응
Level 2 이상의 구체적 성능 수치 주장 측정 조건·환경·데이터셋 한계 명시 §5.6(a) — 오해 방지
비교 클레임 (§7.4 적용) 비교 대상·기준·동일성 조건 명시 §5.6(b) — 비교 맥락
기밀 정보에 근거한 주장 (GAP-08) 기밀 여부 명시 + 검증 한계 공개 §5.6(c) — 검증 한계
라이프사이클 일부 단계만 적용 적용 단계·미적용 단계 및 이유 명시 §5.6(d) — 범위 제한
통계적 불확실성 포함 결과 신뢰 구간·표준 오차·샘플 크기 명시 §5.6(e) — 불확실성
YAML 추가 필드 예시:
explanatory_statement: "본 정확도 수치는 MIMIC-IV 검증 데이터셋(n=10,432) 기반 측정값이며, 실 임상 환경에서는 결과가 상이할 수 있음. 95% CI: [0.91, 0.96]."

S16.3 — Type II 클레임 YAML 템플릿

아래 YAML 템플릿은 ISO/IEC 42117-1 §7.1.2 a–q 요건을 모두 충족하도록 설계된 Type II 자기선언 클레임의 공식 구조입니다. 굵게 표시된 필드는 필수(SHALL)입니다.

# ISO/IEC 42117-X Type II Self-declared Trustworthiness Fact Claim # Reference: ISO/IEC TS 42117-1 §5.2, §7.1.2 (a-q), §7.5 # Template version: 1.0 | Guide: TFL S16 # ================================================================ metadata: claim_id: "SC-[CHAR]-[NNN]" # e.g. SC-Accuracy-001 claim_type: "Type II" # §5.2 Self-declared issued_by: "[Organization Name]" # 선언 주체 조직명 issue_date: "YYYY-MM-DD" valid_until: "YYYY-MM-DD" # §7.1.2.q 재평가 일정 version: "v[X.Y]" product: # §7.1.2.c,d — 제품 식별 및 범위 name: "[AI Product Name]" version: "[semantic version]" type: "system | model | service" scope: "[intended use context and deployment domain]" claim: # §7.1.2.e,f — 구체적·투명한 진술 characteristic: "[TCM Characteristic]" # 28개 TCM 특성 중 하나 level: 1 | 2 # Level 3은 Type I/III 권고 claim_text: "[specific, measurable fact statement]" metric: "[measurement metric name]" threshold: "[target value or range]" actual_result: "[measured result value]" # §7.1.2.b — 검증 가능 applicable_to: "whole_system | component | dataset" # §7.1.2.d evidence: # §7.1.2.f, §7.5.2.f — 방법론 투명성 standard_ref: - "[ISO/IEC standard:year, clause]" evaluation_method: "[test method description]" evaluation_date: "YYYY-MM-DD" dataset_description: "[test dataset spec — if data-dependent]" # §7.5.2.f tool_or_framework: "[evaluation tool/framework]" lifecycle: # §7.1.2.h, §7.5.2.e stages_covered: - "design | development | testing | deployment | operation" uncovered_stages: "[미포함 단계 + 이유]" # §7.5.2.e — 필수 명시 limitations: # §7.1.2.a,k,l — 정확성 보증 known_exclusions: "[클레임이 포함하지 않는 범위]" contextual_constraints: "[지역·언어·도메인 제약]" validity_conditions: "[클레임이 유효한 조건]" supporting_info: # §7.5.2 location: "[URL or QR code — 무료 공개 접근]" # §7.5.2.a programme_operator: "[TFL programme operator, if any]" # §7.5.2.c communication_type: "D2D | B2B | B2C" # §7.5.2.d
📌
필드 작성 우선순위
최소 필수 필드: metadata (claim_id, claim_type, issued_by, issue_date) + product (name, version, type) + claim (characteristic, claim_text, metric, threshold, actual_result) + evidence (standard_ref, evaluation_date)
권장 추가: lifecycle + limitations + supporting_info.location

S16.4A — 평가 방법론 문서화 요건 REC-S16-01 · ISO 14021 §6.2 대응

ISO 14021 §6.2 보존 요건
ISO 14021은 자기선언 주장의 근거 자료를 클레임 사용 기간 동안 및 그 이후 합리적인 기간 동안 유지할 것을 규정합니다. TFL 가이드라인으로는 valid_until 만료 후 최소 3년 보존을 권고합니다. 기밀 정보가 포함된 근거 자료는 독립 검증 기관에 의뢰하거나, 기밀 사실과 검증 한계를 공개 지원 정보에 명시해야 합니다.

S16.5 — 금지 사항 및 제한 (§7.2 · §7.3 · §7.4)

§7.2 — 금지 표현 (SHALL NOT)
아래 표현은 AI 시스템·모델이 신뢰할 수 있거나 유익하다는 것을 광범위하게 암시하므로 사용 절대 금지입니다:
ethical AI trustworthy AI responsible AI human-centric safe AI
✅ S03 Simple Claim 77개 전체에서 위 5개 표현 미사용 확인됨
§7.3 — 일반적 신뢰성 진술 제한 (SHALL)
"ethicality" 또는 "sustainability"와 연계하는 광범위한 신뢰성 진술은 다음 경우에만 허용됩니다:
  • §6 요건을 충족한 인증된 프로그램 내에 포함된 경우, 또는
  • 공개적으로 이용 가능한 합의 기반 문서 (published international standard)에 포함된 경우
⚠️ ISO/IEC TS 42117-1은 현재 PWI 준비 단계이므로 "모든 42117-1 신뢰성 요건을 충족한다"는 주장 불가
§7.4 — 비교 진술 요건 (비교 클레임 작성 시 적용)
타 시스템·모델과 비교하는 경우 다음을 준수해야 합니다:
  • §7.4.1: 비교 기준선 명확 식별 — (a) 자사 이전 버전, (b) 타사 AI, (c) 업계 평균/벤치마크
  • §7.4.2: 동일 기능·측정 단위·라이프사이클 단계 기반 비교
  • §7.4.3: 절대적 차이 또는 상대적 차이를 명확히 구분 표현
🚫

# Selected Claim 적용 예시 (S16.3 YAML 템플릿 확장) claim: characteristic: "Privacy" level: 2 selected_claim_labels: # §7.2~§7.18 중 해당 항목 선택 - "7.11 Privacy-preserving" - "7.12 Data minimization" - "7.10 Opt-in data content" claim_text: "[System]은 차분 프라이버시(ε=0.5)를 적용하고 수집 데이터를 [X]개 항목으로 최소화하며, 전체 학습 데이터의 [Y]%가 명시적 동의(opt-in) 기반입니다" actual_result: "ε=0.5, 데이터 항목 [N]개, opt-in 비율 [Y]%"
⚠️
GAP-06 대응 — 자기선언 클레임의 공신력 제한
§7.2~§7.18 선택 클레임은 Type II 자기선언으로, 독립적 검증이 포함되지 않습니다. 고위험 AI 제품(고영향 AI 분야 §33, 최첨단 AI §32)에서는 Type I (제3자 인증) 또는 Type III (독립 V&V)와 병행하여 공신력을 보완할 것을 권고합니다. → S17 Type I, S14 Type III
